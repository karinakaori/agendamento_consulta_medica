# Agendamento de Consulta Médica

## Descrição do projeto

Este repositório apresenta os artefatos da atividade prática da Unidade III, aplicada ao gerenciamento de um projeto de software com apoio de inteligência artificial generativa.

O cenário analisado é o desenvolvimento de um aplicativo móvel para agendamento de consultas médicas. O sistema contempla cadastro de usuários, gestão de agenda médica, agendamento de consultas, notificações e integração com um sistema externo de prontuário.

O projeto está em fase intermediária de desenvolvimento. Parte das funcionalidades já foi implementada, mas existem pontos críticos relacionados à integração externa, mudanças de requisitos e aumento da carga de trabalho da equipe.

## Objetivo do exercício

O objetivo é usar um LLM como apoio para:

- identificar riscos do projeto;
- analisar qualitativamente impactos e fatores condicionantes;
- definir estratégias de resposta aos riscos mais relevantes;
- elaborar uma comunicação objetiva para stakeholders;
- organizar a documentação em Markdown, mantendo rastreabilidade dos artefatos produzidos.

## Organização do repositório

- `riscos/identificacao.md`: lista dos riscos identificados, com descrição e contexto.
- `riscos/analise.md`: análise qualitativa dos riscos, impactos e fatores condicionantes.
- `riscos/respostas.md`: estratégias de resposta, justificativas e ações associadas.
- `comunicacao/status_stakeholders.md`: comunicado objetivo para stakeholders.
- `prompts/prompts_utilizados.md`: exemplos de prompts usados para apoiar a produção dos artefatos.

## Observação sobre uso de IA

A IA generativa foi utilizada como apoio para estruturar ideias, organizar a análise e redigir os artefatos. As decisões finais foram revisadas considerando o contexto do projeto, a criticidade da integração com prontuário e a capacidade da equipe descrita no cenário.
