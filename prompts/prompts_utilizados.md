# Prompts Utilizados

## Prompt para identificacao de riscos

```text
Atue como gerente de projetos de software. Analise o seguinte cenario: um aplicativo movel para agendamento de consultas medicas esta em fase intermediaria de desenvolvimento. O sistema inclui cadastro de usuarios, gestao de agenda medica, agendamento de consultas, notificacoes e integracao com sistema externo de prontuario. A integracao esta instavel por falta de documentacao clara da API e mudancas recentes no sistema externo. Stakeholders solicitaram alteracoes no fluxo de agendamento. A equipe possui 4 desenvolvedores e 1 tester e relatou aumento da carga de trabalho. Liste riscos do projeto, descreva cada risco e indique o contexto em que podem ocorrer.
```

## Prompt para analise qualitativa

```text
Com base nos riscos identificados para o projeto de aplicativo de agendamento de consultas medicas, faca uma analise qualitativa considerando probabilidade, impacto, prioridade, impactos no projeto e fatores condicionantes. Organize a resposta em uma tabela clara.
```

## Prompt para estrategias de resposta

```text
Para os riscos mais relevantes do projeto, defina estrategias de resposta usando as categorias evitar, mitigar, transferir ou aceitar. Para cada risco, apresente justificativa da escolha e acoes praticas associadas.
```

## Prompt para comunicacao com stakeholders

```text
Elabore uma comunicacao objetiva para stakeholders nao tecnicos sobre a situacao do projeto. A mensagem deve incluir contexto, principais riscos, acoes em andamento, decisoes necessarias e proximos passos. Use linguagem acessivel e foco em tomada de decisao.
```

## Observacao de revisao

As respostas geradas por IA devem ser revisadas por uma pessoa responsavel pelo projeto. O LLM apoia a estruturacao e redacao, mas nao substitui a validacao tecnica, a priorizacao de negocio nem a verificacao de seguranca e privacidade.
