# Prompts Utilizados

## Prompt para identificação de riscos

```text
Atue como gerente de projetos de software. Analise o seguinte cenário: um aplicativo móvel para agendamento de consultas médicas está em fase intermediária de desenvolvimento. O sistema inclui cadastro de usuários, gestão de agenda médica, agendamento de consultas, notificações e integração com sistema externo de prontuário. A integração está instável por falta de documentação clara da API e mudanças recentes no sistema externo. Stakeholders solicitaram alterações no fluxo de agendamento. A equipe possui 4 desenvolvedores e 1 tester e relatou aumento da carga de trabalho. Liste riscos do projeto, descreva cada risco e indique o contexto em que podem ocorrer.
```

## Prompt para análise qualitativa

```text
Com base nos riscos identificados para o projeto de aplicativo de agendamento de consultas médicas, faça uma análise qualitativa considerando probabilidade, impacto, prioridade, impactos no projeto e fatores condicionantes. Organize a resposta em uma tabela clara.
```

## Prompt para estratégias de resposta

```text
Para os riscos mais relevantes do projeto, defina estratégias de resposta usando as categorias evitar, mitigar, transferir ou aceitar. Para cada risco, apresente justificativa da escolha e ações práticas associadas.
```

## Prompt para comunicação com stakeholders

```text
Elabore uma comunicação objetiva para stakeholders não técnicos sobre a situação do projeto. A mensagem deve incluir contexto, principais riscos, ações em andamento, decisões necessárias e próximos passos. Use linguagem acessível e foco em tomada de decisão.
```

## Observação de revisão

As respostas geradas por IA devem ser revisadas por uma pessoa responsável pelo projeto. O LLM apoia a estruturação e redação, mas não substitui a validação técnica, a priorização de negócio nem a verificação de segurança e privacidade.
